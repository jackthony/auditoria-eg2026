### A2 — Confounder tamaño de mesa

**Hipótesis:** n=230 podría ser mesa atípicamente pequeña donde la varianza binomial produce outliers.

**Ejecución:** Para P(X≥208 | n=230, p=0.10) la varianza es np(1-p)=230×0.109×0.891≈22.4, σ≈4.73. Observado: 208. Desviación: (208−25.1)/4.73 ≈ 38.7σ. Con p0=0.70 la varianza sube a np(1-p)=230×0.70×0.30≈48.3, σ≈6.95; desviación: (208−161)/6.95≈6.8σ. Mesas pequeñas (n<50) producen outliers inocentes; n=230 está en el cuartil superior de tamaño normal de mesa peruana. La varianza extra por tamaño es insignificante frente a 38.7σ.

**Resultado:** Confounder tamaño no opera. n=230 es robusto estadísticamente.

**→ SOBREVIVE**

### A3 — Método alternativo: χ² de homogeneidad vs binomial exacto

**Ejecución independiente:**

Tabla 2×2: Mesa 018146 (208 JPP, 22 no-JPP) vs fondo nacional (1,657,500 JPP, 13,538,745 no-JPP).

```
         JPP        no-JPP      Total
Mesa   208          22           230
Fondo  1,657,500  13,538,745  15,196,245
Total  1,657,708  13,538,767  15,196,475
```

Proporciones esperadas bajo H0: E(mesa,JPP) = 230 × (1,657,708/15,196,475) = 230 × 0.10909 ≈ 25.09.

Residuo Pearson para celda (mesa,JPP): (208−25.09)/√25.09 ≈ 182.91/5.009 ≈ 36.52.

χ²_Pearson ≈ 36.52² + contribuciones restantes ≈ 1,333.7 (1 gl).

P(χ²(1) > 1333) → p ≈ 10^(−290) por aproximación cola chi-cuadrado (usando ln P ≈ −χ²/2 = −666.8, log10 ≈ −289.6).

Binomial exacto reportado: log10_p = −170.8. El χ² da valor aún más extremo (diferencia: χ² asintótico vs exacto binomial; en colas tan extremas el exacto es más conservador). **Ambos métodos concuerdan en rechazo masivo de H0; el binomial exacto es el más conservador de los dos.** Orden de magnitud: ambos >>10^−100.

**→ COINCIDE. SOBREVIVE**

### A4 — Verificación numérica independiente (DuckDB/manual)

**Ejecución:**

```sql
-- Verificación phat
SELECT 208.0 / 230.0 AS phat;
-- = 0.904347826...  ✓ coincide con 0.904348

-- Verificación z-score
-- z = (phat - p0) / sqrt(p0*(1-p0)/n)
-- = (0.904348 - 0.109073) / sqrt(0.109073 * 0.890927 / 230)
-- = 0.795275 / sqrt(0.097202 / 230)
-- = 0.795275 / sqrt(0.000422618)
-- = 0.795275 / 0.020558
-- = 38.685...  ✓ coincide con z=38.69

-- Cohen h
-- h = 2*arcsin(sqrt(phat)) - 2*arcsin(sqrt(p0))
-- = 2*arcsin(sqrt(0.904348)) - 2*arcsin(sqrt(0.109073))
-- arcsin(sqrt(0.904348)) = arcsin(0.95097) = 1.2527 rad
-- arcsin(sqrt(0.109073)) = arcsin(0.33027) = 0.33625 rad
-- h = 2*(1.2527 - 0.33625) = 2*0.91645 = 1.8329
-- Reportado: 1.8396 → diferencia 0.0067 (<0.4%)
```

Diferencia en Cohen h: 1.8329 vs 1.8396 — discrepancia del 0.36%, atribuible a precisión en arcsin. No material; ambos clasifican como "muy grande" (h>>0.8).

p-value: no recalculable manualmente para 1.6e-171, pero logsf está documentado correctamente (scipy.stats.binom.logsf evita catastrophic cancellation). Log10_p=−170.8 → consistente con el valor reportado.

**Resultado:** Cálculos verificados. Única discrepancia <0.4% en Cohen h, no material.

**→ OK. SOBREVIVE**

### A6 — Post-hoc fishing check

**Hipótesis:** Mesa 018146 identificada ex-post en búsqueda exhaustiva de todas las mesas → requiere corrección múltiple Bonferroni.

**Análisis:**

El finding declara explícitamente: *"Test post-hoc: mesa identificada por búsqueda exhaustiva"* y aplica contra-argumento. Evalúo independientemente:

- Universo: 78,605 mesas.
- α corregido Bonferroni: 0.001/78,605 = 1.27e-8.
- p reportada: 1.60e-171.
- ¿p_corr < α_Bonferroni? 1.60e-171 << 1.27e-8. **Pasa Bonferroni con 163 órdenes de magnitud de margen.**

La mesa es genuinamente la única con n>100 y pct≥90% entre 78,605 (reportado en finding). Si hubiera 1,000 mesas similares, Bonferroni seguiría siendo 10^−168 vs 10^−8. El fishing no destruye el finding.

**Limitación real:** El finding correctamente declara la búsqueda post-hoc. La corrección Bonferroni está matemáticamente satisfecha. Transparencia presente.

**→ SOBREVIVE** (post-hoc declarado + corrección múltiple superada)
