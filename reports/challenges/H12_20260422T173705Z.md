### A3 — Método alternativo: χ² homogeneidad vs binomial exacto

**Verificación independiente:**

El binomial exacto testa P(X≥208|n=230, p0=0.109073). El equivalente χ² de bondad de ajuste:

```
Observado:  [208, 22]   (JPP, no-JPP)
Esperado:   [25.09, 204.91]  (n×p0, n×(1-p0))

χ² = (208-25.09)²/25.09 + (22-204.91)²/204.91
   = (182.91)²/25.09 + (-182.91)²/204.91
   = 33,476.5/25.09 + 33,476.5/204.91
   = 1,334.3 + 163.4
   = 1,497.7   (1 df)
```

p-value χ²(1497.7, df=1): astronomicamente menor que 10⁻¹⁰⁰. Orden de magnitud: `log10_p ≈ -326` (χ² escala diferente al binomial; ambos dan p < 10⁻¹⁰⁰). El z-score declarado z=38.69 → z²=1,496.9 ≈ χ²=1,497.7. **Coincidencia exacta** (diferencia < 0.1%).

**→ COINCIDE.** Binomial exacto y χ²/z-test consistentes. SOBREVIVE.

### A4 — Verificación numérica independiente (DuckDB/aritmética directa)

**Re-cálculo manual:**

```sql
-- Parámetros
p0    = 0.109073
n     = 230
k_obs = 208
phat  = 208/230 = 0.904348  ✓ (declarado 0.904348)

-- z-test 1-prop
SE    = sqrt(0.109073 * (1-0.109073) / 230)
      = sqrt(0.109073 * 0.890927 / 230)
      = sqrt(0.097196 / 230)
      = sqrt(0.000422593)
      = 0.020557

z     = (0.904348 - 0.109073) / 0.020557
      = 0.795275 / 0.020557
      = 38.685  ≈ 38.69 ✓

-- Cohen h
h = 2*arcsin(sqrt(0.904348)) - 2*arcsin(sqrt(0.109073))
  = 2*arcsin(0.95097) - 2*arcsin(0.33027)
  = 2*(1.25365) - 2*(0.33640)
  = 2.50730 - 0.67280
  = 1.8345  ≈ 1.8396 (diferencia 0.27% — rounding en arcsin) ✓

-- Bonferroni check
α_corregido = 0.05 / 78605 = 6.36e-7
p_mesa = 1.60e-171  <<  6.36e-7  ✓
```

**Diferencia Cohen h:** 1.8345 vs 1.8396 → Δ=0.003 (0.3%). Irrelevante. Clasificación "muy grande >0.8" se mantiene.

**→ VERIFICADO.** Sin error numérico material. SOBREVIVE.

**Conclusión:** Efecto destruído solo si se asume p0 > 87%, lo cual presupone la anomalía. Con cualquier p0 ≤ 70% la conclusión es invariante.

**→ SOBREVIVE** (condicionado a p0 ≤ 70%).
